## <a name="safe-typing-enabled"></a>啟用 「 安全輸入  
 下表顯示受到安全輸入 Oracle 資料類型如何變更的時機**EnableSafeTyping**屬性的繫結 **，則為 true**。  

> [!NOTE]
>  不在此資料表中的 oracle 資料類型會顯示相同的方式，啟用或停用安全的輸入。  

|Oracle 資料類型|XSD 類型|.NET 類型|註解|  
|----------------------|--------------|---------------|-------------|  
|date|xsd:string|String|值應該以 Oracle NLS_DATE_FORMAT。 您可以指定的格式中的日期資料類型**DateFormat**下方的內容繫結**MlsSettings**繫結屬性。 如果未不指定任何值，如**DateFormat**繫結屬性，配接器使用 MLS 設定 ODP.NET 用戶端在同一部電腦上安裝配接器的位置。|  
|TimeStamp|xsd:string|String|值應該以 Oracle NLS_TIMESTAMP_FORMAT。 您可以指定時間戳記中的資料類型的字串格式**TimeStampFormat**下方的內容繫結**MlsSettings**繫結屬性。 如果未不指定任何值，如**TimeStampFormat**繫結屬性，配接器使用 MLS 設定 ODP.NET 用戶端在同一部電腦上安裝配接器的位置。|  

> [!IMPORTANT]
>  如果已啟用安全的輸入，Oracle 數值資料類型資料集和弱型別 REF 資料指標內會一律顯示為字串。  

## <a name="validation"></a>驗證  
 [!INCLUDE[adapteroraclebusinessshort](../../includes/adapteroraclebusinessshort-md.md)]不會執行明確驗證您指定的 Oracle 資料類型的值。 不過，根據 Oracle 資料型別和安全的輸入是啟用還是停用，可能會執行隱含的驗證：  

-   在還原序列化之間的 XML 傳遞訊息和配接器會在內部使用的.NET 類型中。  

-   由 ODP.NET 某些資料類型。
