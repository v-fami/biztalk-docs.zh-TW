# <a name="soa-governance-integration"></a>SOA 控管整合
企業級應用程式必須支援穩定且可靠的管理功能，能夠以符合商務需求、 政府法規、 服務等級協定 (Sla)，以及客戶及交易夥伴的期望。 執行階段控管著重於特別的挑戰之一和需求，成功地執行服務導向架構 (SOA) – 架構的系統符合這些需求。 商務系統所傳遞品質是服務的定義其成功或失敗的主要因素。  

## <a name="soa-challenges"></a>SOA 挑戰  
 SOA 型系統部署到實際執行環境的企業面臨一些挑戰，如下所示：  
  
-   維護和升級的成本降至最低，並允許累加式更新  
  
-   允許透過商務程序管理和撰寫工具快速變更  
  
-   端對端安全性。這包括信任和保護訊息的寄件者、 接收者，與內容的隱私權。  
  
-   識別、 管理和修復發生的例外狀況  
  
-   分開處理服務和取用者  
  
-   測量，並證明位移成本考量 SOA 應用程式的商務價值  
  
-   控制重複或否則不需要服務的激增 （控管）  
  
-   促進潛在使用者初始的開發成本的降低所需的適當服務識別  
  
-   管理服務的成本與持續性的維護的風險降至最低，並變更生命週期  
  
-   簡化的實際使用適當的服務 （分離位置、 傳輸、 原則、 標準和傳訊樣式）  
  
-   報告功能用來識別誰正在使用哪一種服務，，和原因
