# <a name="single-sign-on-support-for-the-wcf-adapters"></a><span data-ttu-id="4446f-102">WCF 配接器的單一登入支援</span><span class="sxs-lookup"><span data-stu-id="4446f-102">Single Sign-On Support for the WCF Adapters</span></span>
<span data-ttu-id="4446f-103">您可以使用 BizTalk 管理主控台來設定「企業單一登入」(SSO)，以便與 WCF 接收位置或傳送埠搭配使用。</span><span class="sxs-lookup"><span data-stu-id="4446f-103">You can configure Enterprise Single Sign-On (SSO) for use with a WCF receive location or send port by using the BizTalk Administration console.</span></span> <span data-ttu-id="4446f-104">本主題將說明 SSO 如何與 WCF 配接器搭配使用。</span><span class="sxs-lookup"><span data-stu-id="4446f-104">This topic describes how SSO works with the WCF adapters.</span></span>  
  
 <span data-ttu-id="4446f-105">在使用者會與各式各樣的系統和應用程式互動的企業環境中，這個環境不大可能會針對多重程序、產品和電腦的整個期間維護使用者的內容。</span><span class="sxs-lookup"><span data-stu-id="4446f-105">In an enterprise environment, where a user interacts with various systems and applications, it is likely that the environment does not maintain the user context through multiple processes, products, and computers.</span></span> <span data-ttu-id="4446f-106">這類使用者內容是提供單一登入能力的重要資訊，因為在確認是誰發出原始要求時必須用到這份資訊。</span><span class="sxs-lookup"><span data-stu-id="4446f-106">This user context is crucial to providing single sign-on capabilities, because it is necessary to verify who initiated the original request.</span></span> <span data-ttu-id="4446f-107">為了解決這個問題，「企業單一登入」(SSO) 提供了 SSO 票證 (非 Kerberos 票證)，應用程式可以用來取得對應提出原始要求之使用者的認證。</span><span class="sxs-lookup"><span data-stu-id="4446f-107">To overcome this problem Enterprise Single Sign-On (SSO) provides an SSO ticket (not a Kerberos ticket) that applications can use to get the credentials that correspond to the user who made the original request.</span></span>  
  
 <span data-ttu-id="4446f-108">當接收配接器收到訊息時，該配接器可能會要求來自 SSO 伺服器的 SSO 票證。</span><span class="sxs-lookup"><span data-stu-id="4446f-108">When a receive adapter gets a message, the adapter can request an SSO ticket from an SSO server.</span></span> <span data-ttu-id="4446f-109">這個加密票證包含發出要求和逾時期限之使用者的 [!INCLUDE[btsWinNoVersion](../includes/btswinnoversion-md.md)] 識別。</span><span class="sxs-lookup"><span data-stu-id="4446f-109">This encrypted ticket contains the [!INCLUDE[btsWinNoVersion](../includes/btswinnoversion-md.md)] identity of the user that made the request and a time-out period.</span></span> <span data-ttu-id="4446f-110">取得票證之後，票證會當成屬性新增至內送訊息。</span><span class="sxs-lookup"><span data-stu-id="4446f-110">After the ticket is acquired, it is added as a property to the incoming message.</span></span> <span data-ttu-id="4446f-111">當傳送配接器傳輸訊息時，配接器會在連絡 SSO 伺服器時使用已發出的 SSO 票證，以及配接器將用來擷取認證的分支機構應用程式名稱。</span><span class="sxs-lookup"><span data-stu-id="4446f-111">When a send adapter transmits a message, the adapter contacts an SSO server with the issued SSO ticket and an affiliate application name for which the adapter is trying to retrieve a credential.</span></span> <span data-ttu-id="4446f-112">SSO 伺服器會查詢目標分支機構應用程式的使用者認證，然後將該認證傳回給傳送配接器，由它使用認證，將已經過適當驗證的訊息傳送到該分支機構應用程式。</span><span class="sxs-lookup"><span data-stu-id="4446f-112">The SSO server looks up the user credential for the target affiliate application, and then returns the credential to the send adapter, which uses it to send an appropriately authenticated message to the affiliate application.</span></span>
