## <a name="x12-and-edifact"></a><span data-ttu-id="96543-126">X12 和 EDIFACT</span><span class="sxs-lookup"><span data-stu-id="96543-126">X12 and EDIFACT</span></span>  
 <span data-ttu-id="96543-127">ANSI X12 和 UN/EDIFACT 這兩項標準構成超過 90% 的所有全球交換 EDI 訊息：</span><span class="sxs-lookup"><span data-stu-id="96543-127">Two standards, ANSI X12 and UN/EDIFACT, constitute more than 90% of all EDI messages exchanged globally:</span></span>  
  
- <span data-ttu-id="96543-128">UN/EDIFACT EDI 國際訊息標準 (用於行政、商業、貿易的 EDI) 是由「聯合國歐洲經濟委員會」(UN/ECE) 所開發並持續維護，</span><span class="sxs-lookup"><span data-stu-id="96543-128">The UN/EDIFACT EDI international message standard (EDI for Administration, Commerce, and Trade) was developed and continues to be maintained by the United Nations Economic Commission for Europe (UN/ECE).</span></span> <span data-ttu-id="96543-129">也稱為 EDIFACT。</span><span class="sxs-lookup"><span data-stu-id="96543-129">Also known as just EDIFACT.</span></span>  
  
- <span data-ttu-id="96543-130">X12 EDI U.S. 訊息標準是由「美國國家標準局」(ANSI) 授權的 Accredited Standards Committee (ASC) X12 委員會所開發並持續維護。</span><span class="sxs-lookup"><span data-stu-id="96543-130">The X12 EDI U.S. message standard was developed and continues to be maintained by the Accredited Standards Committee (ASC) X12 committee chartered by the American National Standards Institute (ANSI).</span></span>  
  
  <span data-ttu-id="96543-131">EDIFACT 很大部分基於美國X12 標準。</span><span class="sxs-lookup"><span data-stu-id="96543-131">EDIFACT is based largely on the U.S. X12 standards.</span></span> <span data-ttu-id="96543-132">這兩項 EDI 標準在結構上非常類似，</span><span class="sxs-lookup"><span data-stu-id="96543-132">The two EDI standards are very similar in terms of structure.</span></span> <span data-ttu-id="96543-133">差異包括：</span><span class="sxs-lookup"><span data-stu-id="96543-133">Differences include:</span></span>  
  
- <span data-ttu-id="96543-134">這兩項標準的結構化元素名稱不同。</span><span class="sxs-lookup"><span data-stu-id="96543-134">Structural elements are named differently in the two standards.</span></span> <span data-ttu-id="96543-135">例如，交換控制標頭在 X12 中是 ISA 資料元素，而在 EDIFACT 中是 UNB 資料元素。</span><span class="sxs-lookup"><span data-stu-id="96543-135">For example, the interchange control header is an ISA data element in X12 and a UNB data element in EDIFACT.</span></span>  
  
- <span data-ttu-id="96543-136">儘管這兩項標準的許多交易集相對應，但這兩項標準的交易集名稱不同。</span><span class="sxs-lookup"><span data-stu-id="96543-136">While many transaction sets map between the two standards, the transaction sets are named differently in the two standards.</span></span> <span data-ttu-id="96543-137">例如，訂單在 X12 中是 850 交易集，而在 EDIFACT 中是 ORDERS 交易集。</span><span class="sxs-lookup"><span data-stu-id="96543-137">For example, a purchase order is an 850 transaction set in X12 and an ORDERS transaction set in EDIFACT.</span></span>  
  
- <span data-ttu-id="96543-138">EDIFACT 有選擇性的標頭區段 UNA，可設定結構化元素例如分隔符號和小數點標記，而覆寫管線中定義的預設值。</span><span class="sxs-lookup"><span data-stu-id="96543-138">EDIFACT has an optional header segment UNA to set structural elements such as separators/delimiters and decimal notation, overriding the default values defined in a pipeline.</span></span>  
  
- <span data-ttu-id="96543-139">X12 有兩個通知 (名為 TA1 的技術通知和名為 997 的功能通知)，而 EDIFACT 有一個名為 CONTRL 的綜合性通知。</span><span class="sxs-lookup"><span data-stu-id="96543-139">X12 has two acknowledgments (a technical acknowledgment called TA1 and a functional acknowledgment called 997), while EDIFACT has one comprehensive acknowledgment called CONTRL.</span></span>  
  
- <span data-ttu-id="96543-140">X12 建議使用 ASCII 編碼，而 EDIFACT 建議使用 UTF8 編碼。</span><span class="sxs-lookup"><span data-stu-id="96543-140">X12 recommends ASCII encoding, while EDIFACT recommends UTF8 encoding.</span></span>  
  
  <span data-ttu-id="96543-141">BizTalk Server 支援 KEDIFACT (韓文 EDIFACT) 標準。</span><span class="sxs-lookup"><span data-stu-id="96543-141">BizTalk Server supports the KEDIFACT (Korea EDIFACT) standard.</span></span> <span data-ttu-id="96543-142">KEDIFACT 遵循「UN/EDIFACT 的訊息實作指南」，因此大部分是以 EDIFACT 為基礎。</span><span class="sxs-lookup"><span data-stu-id="96543-142">KEDIFACT follows the Message Implementation Guide of UN/EDIFACT, so is closely based on EDIFACT.</span></span> <span data-ttu-id="96543-143">不過，KEDIFACT 和 X12/EDIFACT 之間還是有差異。</span><span class="sxs-lookup"><span data-stu-id="96543-143">However, there are differences between KEDIFACT and X12/EDIFACT.</span></span> <span data-ttu-id="96543-144">KEDIFACT 使用數個 KEDIFACT 特有的 EDI 結構描述並使用 KECA 字碼頁。</span><span class="sxs-lookup"><span data-stu-id="96543-144">KEDIFACT uses several KEDIFACT-specific EDI schemas and uses the KECA code page.</span></span>
