# <a name="viewing-tracked-message-and-instance-data"></a>檢視追蹤的訊息和執行個體資料
您可以使用歷程記錄和追蹤資料來協助疑難排解 BizTalk Server 應用程式。 例如，系統管理員可以使用歷程記錄和追蹤資料：  
  
-   檢視追蹤的訊息事件、 訊息屬性和訊息內文在訊息流程的各個階段。 此資料可用於疑難排解或稽核用途。  
  
-   重新執行特定的協調流程執行個體。  
  
-   追蹤規則引發事件，以重建在稍後的事件順序的時間。  
  
-   藉由指定如訊息結構描述和訊息屬性/值組等準則的訊息查詢。 藉此找出訊息之後，使用者可判斷訊息已進展至訊息流程的哪一個點。 進階使用者也可以建立訊息的自訂 SQL Server 查詢。  
  
-   搜尋封存資料庫中的封存資料。
