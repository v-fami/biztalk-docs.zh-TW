# <a name="source-control-and-the-biztalk-project-system"></a>原始檔控制和 BizTalk 專案系統
原始檔控制是一種機制，軟體中央部分可在其中儲存和追蹤檔案版本以及控制可存取檔案的使用者。 當您使用原始檔控制時，可以：  
  
-   限制覆寫重要檔案的可能性。  
  
-   將版本號碼套用到檔案。  
  
-   封存舊版之原始檔控制的檔案。  
  
-   追蹤修改檔案的使用者、修改的時間以及修改的內容。  
  
 Visual Studio 可讓您與所有 Microsoft 原始程式碼控制介面 (MSSCCI) 相容的系統整合。 [!INCLUDE[btsBizTalkServer2006r3](../includes/btsbiztalkserver2006r3-md.md)]提供 Team Foundation Server 的支援，可讓開發團隊充分運用整合式原始檔控制、 錯誤追蹤、 團隊開發支援、 與 Microsoft Office Project Server，並支援使用自動化組建整合MSBuild。 BizTalk 專案系統也支援 Microsoft Visual SourceSafe。  
  
 如需原始檔控制的詳細資訊，請參閱 < 原始檔控制基本概念"Visual Studio 組合集合中在[http://go.microsoft.com/fwlink/?LinkId=25506](http://go.microsoft.com/fwlink/?LinkId=25506)。
