# <a name="understanding-tracked-data"></a>瞭解追蹤資料
當您執行的追蹤查詢時，追蹤的資訊會出現在 [查詢結果] 視窗底部的 [結果] 清單中。  
  
## <a name="viewing-message-details"></a>檢視訊息詳細資訊  
 您可以追蹤訊息屬性。 您也可以追蹤的訊息內文儲存至檔案，並檢視使用非 Microsoft 工具。  
  
-   您可使用滑鼠右鍵按一下服務執行個體所參考的任何訊息，然後選取 [訊息詳細資料]。  
  
-   如果已處理訊息，但追蹤，因為您所開啟的追蹤，您可以將它儲存到您的硬碟，並檢查它。  
  
-   您可以連接至協調流程執行個體，並使用「協調流程偵錯工具」。  
  
## <a name="viewing-service-events"></a>檢視服務事件  
 當已擱置的服務會出現在事件記錄檔時，您可以藉由調查服務**訊息流程**，**協調流程偵錯工具**，**顯示追蹤的訊息事件**，**顯示即時服務執行個體**，從選項**內容**功能表。  
  
## <a name="viewing-orchestration-events"></a>檢視協調流程事件  
 使用協調流程偵錯工具來檢視訊息執行個體透過協調流程取得的路徑。 當您進行時，協調流程的影像會顯示訊息的進度，讓您在協調流程中放置中斷點以進行偵錯。
