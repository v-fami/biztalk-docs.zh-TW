## <a name="introduction-to-as2"></a><span data-ttu-id="14a03-104">AS2 簡介</span><span class="sxs-lookup"><span data-stu-id="14a03-104">Introduction to AS2</span></span>  
 <span data-ttu-id="14a03-105">EDI 所使用的一般傳輸方法是加值型網路 (VAN)。</span><span class="sxs-lookup"><span data-stu-id="14a03-105">A common transport used for EDI is Value-Added Networks (VANs).</span></span> <span data-ttu-id="14a03-106">這類私用網路可提供加值型服務，例如精確與合法的繫結稽核線索。</span><span class="sxs-lookup"><span data-stu-id="14a03-106">These are private networks that provide value-added services, such as precise and legally binding audit trails.</span></span> <span data-ttu-id="14a03-107">然而，許多公司是透過網際網路移轉來交換 EDI 文件。</span><span class="sxs-lookup"><span data-stu-id="14a03-107">However, companies are migrating to exchanging EDI documents over the Internet.</span></span> <span data-ttu-id="14a03-108">這種做法可降低成本、增加彈性與效率，而且就重複性和擴充性而言也具有優勢。</span><span class="sxs-lookup"><span data-stu-id="14a03-108">This lowers costs, increases flexibility and efficiency, and has advantages in terms of redundancy and scalability.</span></span>  
  
 <span data-ttu-id="14a03-109">透過網際網路傳輸 EDI (EDIINT) 的最常見實作方式，就是使用 AS2 (Applicability Statement 2)。</span><span class="sxs-lookup"><span data-stu-id="14a03-109">The most common way of implementing EDI over the Internet (EDIINT) is by AS2 (Applicability Statement 2).</span></span> <span data-ttu-id="14a03-110">AS2 規格定義了以 MIME 為基礎的安全點對點商務資料交換。</span><span class="sxs-lookup"><span data-stu-id="14a03-110">The AS2 specification defines MIME-based secure peer-to-peer business data interchange.</span></span> <span data-ttu-id="14a03-111">訊息 (包含具有 MIME 資料的信封) 的傳輸是使用 HTTP over TCP/IP。</span><span class="sxs-lookup"><span data-stu-id="14a03-111">Messages containing an envelope with MIME data are transmitted using HTTP over TCP/IP.</span></span>  
  
 <span data-ttu-id="14a03-112">AS2 使用 HTTP POST 作業來傳送 EDI、XML 或其他商務資料。</span><span class="sxs-lookup"><span data-stu-id="14a03-112">AS2 uses the HTTP POST operation to send EDI, XML, or other business data.</span></span> <span data-ttu-id="14a03-113">AS2 並沒有限制為僅傳送 EDI 資料。</span><span class="sxs-lookup"><span data-stu-id="14a03-113">AS2 is not restricted to sending EDI data.</span></span> <span data-ttu-id="14a03-114">Request-URI 會識別用來解開和處理訊息資料的處理程序。</span><span class="sxs-lookup"><span data-stu-id="14a03-114">Request-URI identifies a process to be used to unpack and handle message data.</span></span> <span data-ttu-id="14a03-115">訊息處理通知 (Message Disposition Notification，MDN) 會以通知的形式，透過 HTTP 回應訊息內文傳回，或是透過針對原始傳送者 URL 的新 HTTP POST 作業傳回。</span><span class="sxs-lookup"><span data-stu-id="14a03-115">A message disposition notification (MDN) is returned as an acknowledgment either in the HTTP response message body or by a new HTTP POST operation to a URL for the original sender.</span></span>  
  
 <span data-ttu-id="14a03-116">如需有關 EDI 傳訊的詳細資訊，請參閱[AS2 傳訊](../core/as2-messaging.md)。</span><span class="sxs-lookup"><span data-stu-id="14a03-116">For more information about EDI messaging, see [AS2 Messaging](../core/as2-messaging.md).</span></span>
