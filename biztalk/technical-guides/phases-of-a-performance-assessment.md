# <a name="phases-of-a-performance-assessment"></a>效能評定的階段
其中一個主要目標的[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]是為配合多個處理的案例中，盡可能提供最大的彈性。 這很大的彈性，因為 BizTalk 解決方案的開發人員所面臨的主要挑戰之一是要判斷如何發揮最大功能用於[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]依據其業務的需求。 最佳化 BizTalk Server 解決方案的效能時，這種彈性也會帶來一項挑戰。  
  
 本章節描述應該用來將 BizTalk Server 解決方案的效能最佳化參與 BizTalk Server 效能評定的方法。 BizTalk Server 效能評定用來將特定的 BizTalk Server 解決方案的效能最大化。 若要取得最大的好處，從 BizTalk Server 效能評定，效能評定應該分成下列五個主要步驟/階段：  
  
1. 範圍。  
  
2. 計畫  
  
3. 準備  
  
4. 建置實驗室  
  
5. Execute  
  
   本主題會描述每一個階段中有更詳細。

   BizTalk Server 效能評定的階段  
  
## <a name="in-this-section"></a>本節內容  
  
-   [第 1 階段： 設定評量的範圍](../technical-guides/phase-1-scoping-the-assessment.md)-說明建立效能評定的範圍時，應遵循的步驟。  
  
-   [第 2 階段： 規劃評量](../technical-guides/phase-2-planning-the-assessment.md)– 說明如何建立方案的里程碑時間軸。 這用作為主要計劃來清楚文件時應該符合預期的目標方案的測試。  
  
-   [第 3 階段： 準備評量](../technical-guides/phase-3-preparing-for-the-assessment.md)– 說明如何提供詳細的架構圖的解決方案和解決方案所使用的硬體組態的特定層面。  
  
-   [階段 4： 建置評量環境](../technical-guides/phase-4-building-the-assessment-environment.md)– 說明硬體和根據詳細的設計先前建立的方案平台的應用程式的安裝。  
  
-   [第 5 階段： 執行評量](../technical-guides/phase-5-executing-the-assessment.md)– 提供產生透過自動化的負載測試，以及如何偵測並排除在方案中的任何瓶頸的效能資料的相關資訊。
