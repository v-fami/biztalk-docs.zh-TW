# <a name="how-to-disable-tracking"></a>如何停用追蹤
本主題描述如何停用追蹤使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]管理主控台。 您可以在協調流程的執行階段期間設定追蹤的各種選項，傳送埠、 接收埠及管線使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]管理主控台。 您可以隨時變更任何項目的追蹤選項，而不需要中斷商務程序。  

## <a name="prerequisites"></a>必要條件  
 若要執行這個主題中的程序，您必須使用「BizTalk Server 系統管理員」群組成員的帳戶登入。 如需詳細的權限的相關資訊，請參閱[管理的應用程式的權限](../technical-guides/permissions-for-managing-an-application.md)。  

> [!NOTE]
>  如果您只想檢視的追蹤選項，您可以登入的成員身分[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]操作員 」 群組。  

##  <a name="BKMK_DisableOrchTracking"></a> 若要停用協調流程的追蹤  

1. 按一下 **開始**，按一下**所有程式**，按一下  **Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然後按一下**BizTalk Server 管理**。  

2. 在主控台樹狀目錄中，依序展開[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)] **Administration**，展開 BizTalk 群組，展開**應用程式**，然後展開包含您想要在協調的流程的應用程式設定追蹤。  

3. 按一下 **協調流程**，從右窗格中，以滑鼠右鍵按一下您要設定追蹤，然後按一下協調流程**屬性**。  

4. 按一下 **追蹤**索引標籤上，停用追蹤選項下, 表中所述，然後按一下**確定**。  

   |使用|以進行此動作|  
   |--------------|----------------|  
   |**追蹤事件-協調流程開始和結束**|清除此核取方塊，以停用的協調流程執行個體的追蹤之前, 和之後的整個商務程序的處理。|  
   |**追蹤事件-訊息傳送與接收**|清除此核取方塊，以停用追蹤訊息的傳送與接收事件。 此核取方塊才會提供您所選取**協調流程開始和結束**核取方塊。|  
   |**追蹤事件-流程化開始與結束**|當您不需要偵錯協調流程偵錯工具中的協調流程執行個體時，請清除此核取方塊。 當您選取此核取方塊，會填入此「協調流程偵錯工具」中的事件清單。 此核取方塊才會提供您所選取**協調流程開始和結束**核取方塊。|  
   |**追蹤訊息內文-協調流程處理之前**|清除此核取方塊，以停用儲存和追蹤實際訊息內容，在處理之前的協調流程執行個體。 此核取方塊才會提供您所選取**訊息傳送與接收**核取方塊。|  
   |**追蹤訊息內文-協調流程處理之後**|清除此核取方塊，以停用儲存和處理後的實際訊息內容的協調流程執行個體的追蹤。 此核取方塊才會提供您所選取**訊息傳送與接收**核取方塊。|  
   |**追蹤訊息屬性-連入訊息**|清除此核取方塊，以停用追蹤輸入訊息的升級屬性。 此核取方塊才會提供您所選取**訊息傳送與接收**核取方塊。|  
   |**追蹤訊息屬性-外寄訊息**|清除此核取方塊，以停用追蹤輸出訊息的升級屬性。  此核取方塊才會提供您所選取**訊息傳送與接收**核取方塊。|  

### <a name="to-disable-tracking-for-a-send-port"></a>若要停用傳送埠的追蹤  

1. 按一下 **開始**，按一下**所有程式**，按一下  **Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然後按一下**BizTalk Server 管理**。  

2. 在主控台樹狀目錄中，依序展開[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)] **Administration**，展開 BizTalk 群組，展開**應用程式**，然後展開包含您要設定傳送埠的應用程式追蹤。  

3. 按一下 **傳送埠**，以滑鼠右鍵按一下傳送埠，按一下**屬性**，然後按一下**追蹤**。  

4. 下表中所述，停用追蹤選項，然後按一下 **確定**。  

   |使用|以進行此動作|  
   |--------------|----------------|  
   |**追蹤訊息內文-連接埠處理前要求訊息**|清除此核取方塊可停用儲存和追蹤訊息內容之前收到的訊息。 **注意：** 您必須啟用訊息內文管線追蹤，才能成功追蹤連接埠處理前的回應訊息。|  
   |**追蹤訊息內文-連接埠處理後要求訊息**|清除此核取方塊可停用儲存和追蹤訊息內容收到訊息之後。|  
   |**追蹤訊息內文-連接埠處理前回應訊息**|清除此核取方塊可停用儲存和追蹤訊息內容傳送訊息之前。 此核取方塊只適用於請求-回應傳送埠。|  
   |**追蹤訊息內文-連接埠處理後回應訊息**|清除此核取方塊可停用儲存和追蹤訊息內容傳送訊息之後。 此核取方塊只適用於請求-回應傳送埠。|  
   |**追蹤訊息屬性-連接埠處理前要求訊息**|清除此核取方塊，以停用追蹤輸入訊息的升級屬性。|  
   |**追蹤訊息屬性-連接埠處理後要求訊息**|如果您不想要追蹤輸出訊息升級的屬性，請清除此核取方塊。|  
   |**追蹤訊息屬性-連接埠處理前回應訊息**|清除此核取方塊可停用儲存和追蹤訊息內容傳送訊息之前。 此核取方塊只適用於請求-回應傳送埠。|  
   |**追蹤訊息屬性-連接埠處理後回應訊息**|清除此核取方塊，以停用儲存和傳送訊息之後，追蹤的屬性。 此核取方塊只適用於請求-回應傳送埠。|

### <a name="to-disable-tracking-for-a-policy"></a>若要停用原則的追蹤  

1. 按一下 **開始**，按一下**程式**，按一下  **Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然後按一下**BizTalk Server 管理**。  

2. 在主控台樹狀目錄中，依序展開[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)] **Administration**，展開 BizTalk 群組，展開**應用程式**，然後展開包含您要設定之的原則的應用程式追蹤。  

3. 按一下 **原則**，以滑鼠右鍵按一下原則，按一下**屬性**，然後按一下**追蹤**。  

4. 下表中所述，停用追蹤選項，然後按一下 **確定**。  

   |使用|以進行此動作|  
   |--------------|----------------|  
   |**快速活動**|清除此核取方塊，以停用追蹤原則運作所在的執行個體資料。|  
   |**條件評估**|清除此核取方塊，以停用追蹤的 true/false 條件的結果中選取的原則。|  
   |**規則引發**|清除此核取方塊，以停用追蹤動作的原則所啟動。|  
   |**議程更新**|清除此核取方塊，以停用追蹤議程的更新。 議程包含一份值為 "True" 且需要引發的動作清單。|  

### <a name="to-disable-tracking-for-a-schema"></a>若要停用追蹤結構描述  

1. 按一下 **開始**，按一下**程式**，按一下  **Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然後按一下**BizTalk Server 管理**。  

2. 在主控台樹狀目錄中，依序展開[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)] **Administration**，展開 BizTalk 群組，展開**應用程式**，然後展開包含您要設定的結構描述的應用程式追蹤。  

3. 按一下 **結構描述**，以滑鼠右鍵按一下結構描述，然後按一下**屬性**。  

4. 在左窗格中，按一下**追蹤**。  

5. 執行下列動作，以指定您想要停用來追蹤訊息，哪些屬性其中一個項目，然後按一下**確定**:  

   -   清除**永遠追蹤所有屬性**核取方塊，如果您不會想要使用結構描述版本不限的所有訊息屬性。 此核取方塊僅適用於文件結構描述。  

   -   清除**選取所有訊息屬性**核取方塊，如果您不想要使用所有列出的屬性。  

   -   底下**內容清單**，清除不想使用每個屬性的核取方塊。  

   > [!NOTE]  
   >  因為追蹤訊息會在建立效能和儲存體額外負荷會為您的系統，您應該選取針對您的需求的選項。  

### <a name="to-disable-tracking-for-a-pipeline"></a>若要停用管線的追蹤  

1. 按一下 **開始**，按一下**程式**，按一下  **Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然後按一下**BizTalk Server 管理**。  

   > [!NOTE]  
   >  如果您設定在管線的追蹤選項時，您也會設定每個管線所使用的連接埠的全域追蹤選項。 這樣一來，反而可能會導致遠比您預期，正在追蹤的資料這將會變慢系統效能。 相反地，您可以設定追蹤選項，傳送連接埠和接收埠。  

2. 在主控台樹狀目錄中，依序展開[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)] **Administration**，展開 BizTalk 群組，展開**應用程式**，然後展開包含您要設定的管線的應用程式追蹤。  

3. 執行下列其中之一：  

   -   若要停用追蹤其中一個預設 BizTalk 管線中，展開\<所有成品\>。  

   -   若要停用追蹤已部署到 BizTalk 應用程式的自訂管線，請展開包含管線的應用程式。  

4. 按一下 **管線**資料夾中，管線，以滑鼠右鍵按一下，然後按一下**追蹤**。  

   > [!NOTE]  
   >  停用追蹤多個管線，按住 Shift 鍵，按一下 設定每個管線，其中一個管線，以滑鼠右鍵按一下，然後按一下**追蹤**。  

5. 下表中所述，停用追蹤選項，然後按一下 **確定**。  


   |                使用                |                                                                                                                                                                                                                                                                                                                                 以進行此動作                                                                                                                                                                                                                                                                                                                                 |
   |----------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   |     **連接埠開始與結束事件**      |                                                                                                                                                                                                                                                                  清除此核取方塊，追蹤執行個體的開始和結束時，才停用。 詳細資訊包括項目名稱、組件及其他中繼資料。                                                                                                                                                                                                                                                                  |
   |  **訊息傳送與接收事件**   |                                                                                                                                                                                                                                                      清除此核取方塊，以停用追蹤的訊息傳送與接收事件。 就使用此核取方塊才**連接埠開始與結束事件**已選取。                                                                                                                                                                                                                                                       |
   | **管線處理前的訊息** | 清除此核取方塊可停用儲存及追蹤管線所接收的訊息內文，其中保存中繼資料，例如 Url 及升級屬性。 若此管線屬於接收管線，則訊息內文即為傳輸元件提交給管線的原始訊息。 視您的應用程式而定，訊息可能經過加密、簽章或編碼。 當使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]對應，如果這是接收管線中，會在追蹤之後處理輸入的對應。<br /><br /> 就使用此核取方塊才**訊息傳送與接收事件**已選取。 |
   | **在管線處理後的訊息**  |                        清除此核取方塊可停用儲存及追蹤管線所傳送的訊息內文，其中保存中繼資料，例如 Url 及升級屬性。 若此管線屬於接收管線，則訊息內文即為要提交給 MessageBox 資料庫的已處理訊息，視您的應用程式而定，這個訊息可能是 XML。 當使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]對應，如果這是傳送管線，會在執行追蹤之前處理輸出對應。<br /><br /> 就使用此核取方塊才**訊息傳送與接收事件**已選取。                        |
